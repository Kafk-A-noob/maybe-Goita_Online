# 鉄の掟 (Iron Rules)
ファイル生成、コマンド実行、コード変更を伴うあらゆるタスクにおいて、以下の手順を**物理的に**強制する。

1. **Task.mdの作成 (最優先)**
   タスク開始時、他のあらゆる操作（ファイルの読み込み、作成、コマンド）を行う前に、必ず `task.md` を作成または更新しなければならない。

2. **承認チェックボックスの設置**
   `task.md` のタスクリストの **1行目** は、必ず以下でなければならない。
   `[ ] 実行計画に対するユーザーの承認を得る`

3. **実行ブロック (Blockade)**
   1行目の `[ ]` が `[x]` に変わる（ユーザーが明示的に許可を出す）まで、`write_to_file` や `run_command` などの副作用のあるツール使用は**システム的に禁止**されていると見なす。

4. **違反時の措置**
   もし承認前に手が滑って実行しようとした場合、それは「バグ」である。直ちに停止し、ユーザーに謝罪して承認を求めよ。
